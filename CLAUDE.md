## Design Context

### Users
Potential cloud and DevOps clients, recruiters, and marketplace visitors from Upwork, Fiverr, LinkedIn, and GitHub. They arrive needing confidence that Raj can stabilize, deploy, automate, and hand off production cloud systems without heavy explanation.

### Brand Personality
Premium, technical, direct. The site should feel editorial and precise, with strong cloud infrastructure credibility and a clear path to contact.

### Aesthetic Direction
Monochrome editorial base with restrained, meaningful accents. AWS, Azure, DevOps, CI/CD, security, monitoring, and production handoff should lead. AI, backend, and IoT can appear as supporting capabilities only when tied to cloud delivery.

### Design Principles
- Cloud and DevOps credibility comes first; supporting skills must not overpower that story.
- Use black/white structure, large compressed headings, tight mono labels, and disciplined spacing.
- Add color only when it carries meaning: availability, project identity, credentials, contact hover, or skill category cues.
- Keep motion useful and calm: no stutter, no gimmicks, no layout-shifting count animations.
- Every breakpoint must feel intentionally designed, with no clipped headings, overlap, horizontal overflow, or awkward footer/dock states.
